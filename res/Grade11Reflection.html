Manual G-Code Sphere

I spent my free time outside of school writing a program using the computer programming language C++ that generates the G-code for the toolpath. Most CAM (Computer-Aided Manufacturing) software generates a toolpath using complex algorithms that are not visible to the operator. However, my prior programming skills allowed me to integrate the computer science and manufacturing fields to create a part with a program generated entirely independently. The program uses the principles behind the unit circle to procedurally generate the toolpath using these trigonometric principles. The day following, I informed my Advanced Manufacturing teacher, Mr. Trottier, about the program. To my surprise, he was impressed and told me to run the program on our machines. I immediately knew my small project that I worked on outside of school became a real shop project. At this moment, an HR manager at a local manufacturing company visited and took a look at what the students were working on. After I explained the technical concepts, he was interested and wanted to hire me. The following week, I was hired. When the part finished machining, I was impressed to see a semi-sphere cutout using 3d surfacing on a square part. Unlike the previous implementation we have done as a shop that uses Fusion, this was impressive because it shows that Fusion360 is not 100% necessary (but more efficient) to generate this G-code. Even though this process can be questioned, the concept was essential and was revised later in 12th grade to complete the QR codes project.

Cutter Compensation Project

For this project, we were required to work in groups to create two independent parts that mesh together seamlessly into one part. The purpose of this part was to prove our knowledge on Cutter Compensation. In order for a profile and a pocket to align and mesh without physical restriction, the profile’s radial profile needs to be slightly bigger than the pocket. As this difference needs to be as small as possible to reduce the noticeable gap, cutter compensation was used at the control to control the offset in which the toolpath compensates for the tool, increasing the radial profile size. By default, compensation for the tool is automatically compensated by the G-code program, and is typically hardcoded from the G-code that Fusion360 generates. However, in Fusion360, changing the cutter compensation to “In control” removes all compensation and uses G-code specific cutter compensation commands, such as D and G40/G41/G42. These allow for precise offset control on the Haas machine directly, allowing precise tweaking of the radial size of each profile and the pocket. As a result, the profile and the pocket were machined within a thousandth of an inch, creating a seamless appearance when meshed together. As well as the basic rounded profile in the center of the part, four holes were placed on the side with pegs on the other part; they were machined using the same concept.

Restroom signs

For the administration, Dr. Riccio, I was tasked with creating two restroom signs for each gender out of brass to be used for The Larkin, near and part of Essex North Shore Agricultural & Technical School. Dr. Riccio sent out an ideal design for the sign, probably from the internet, that meets her expectations. To ensure her expectations were met, I applied a simple algorithm to take the image, select the four corners of the sign, and reverse the perspective to straighten out the image; this made image operations and tracing very easy. I uploaded the image into Fusion360, and began tracing the image. I used arc curves and other complex geometry, as well as using a font finder tool to find the exact font from the image. I realized the design was in fact impossible to manufacture. It wasn’t that big of a deal though, because I could make very tiny tweaks to the design to make it manufacturable. The reasoning is, because an end mill can only machine surface where its internal complexity is relative to the tool diameter. To achieve a very high complexity, a tiny end mill needs to be used. For this project, moving the letters slightly ensured that, with the end mills we have, it was manufacturable. The part was manufactured with aluminum first for cost-effective mistakes, but the later trusted program was run on brass for the final product.

4-Axis Part - Haas ST-15Y

One of the final projects we worked on in 11th grade of Advanced Manufacturing was the 4-axis part. 4-axis lathes are essential to modern manufacturing. However, they are complex to a basic machinist. For a 4-axis machine with live tooling, there were radial tools and axial tools. This project involved a drill for a simple axial tool, but had an end mill as a radial tool to create a counterbore and a flat for it to lie on. This process was complex, because it involved figuring out how to use live tooling in Fusion360. However, once the concepts were figured out, it was pretty simple. Me, and everyone else in the shop, programmed the part in Fusion360 and ran it on our Haas ST-15Y 4-Axis live tooling lathe.
